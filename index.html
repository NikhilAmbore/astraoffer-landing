Interviewer question
"Design a notification service that can handle 10 million users."
AstraOffer answer
I'd start with requirements: push, email, SMS. At 10M users, fan-out is the core problem. I'd use Kafka with per-channel consumer groups. Notification preferences sit in Redis for sub-millisecond reads. Email/SMS delegated to SendGrid/Twilio via async workers. Rate-limit per user at the queue layer. Delivery receipts in Cassandra. At 10M that's ~50K events/sec at peak — fully horizontal.
✓ Answer ready · 1.1s

Coaching sheet
Say first: "I'll use a hash map to get O(n) instead of O(n²) brute force."

Code: seen = {}; for i,n in enumerate(nums): if target-n in seen: return [seen[target-n],i]; seen[n]=i

Say while coding: "For each number I check if its complement is in the map…"
Complexity: O(n) time · O(n) space

After download

Setup in 5 minutes.

Follow these steps exactly. Takes one time only.

1
Install the app
Mac: Double-click the .dmg, drag to Applications, eject the DMG.
Windows: Run the .exe installer, launch from the Start menu.
2
Bypass the security warning (one time only)
Mac — "damaged and can't be opened": Open Terminal and run:
xattr -cr "/Applications/AstraOffer Assistant.app"
⚠ One-time only.

Windows — "Windows protected your PC": Click More info → Run anyway.
3
Grant Screen Recording (Mac only)
System Settings → Privacy & Security → Screen Recording → enable AstraOffer Assistant. Restart the app.
4
Sign in with your AstraOffer account
Open the app → enter your email → Continue. No account yet? Create one at astraoffer.org. AI is included — no extra keys needed.
5
Upload your resume and start
Setup → paste your resume + job description → Start Session.
During interviews: Cmd+Shift+S to listen · Cmd+Shift+C to capture a coding problem.
